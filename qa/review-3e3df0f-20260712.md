## Findings

### Non-blocking

1. **Keystroke Validate does not call `maxCFGScale` / full `currentEcosystem` logic** (`ui.go:698–703`). Spec §5b shows preset-AIR match then full AIR parsing; implementation uses:

   ```go
   maxVal := 30.0
   modelVal := strings.ToLower(inputs[fiModel].Value())
   if strings.Contains(modelVal, "flux") {
       maxVal = 7.0
   }
   ```

   Functionally correct for all four stock presets and any AIR containing `"flux"`. Submit path uses the proper helper. Prefer unifying on `maxCFGScale` + shared eco detection so keystroke and submit never diverge (e.g. a non-flux AIR that still needs a special max, or future ecosystems).

2. **`currentEcosystem` AIR fallback slightly narrower than §2** — omits explicit `flux-klein` and `stable-diffusion-1` substrings. `klein` still covers flux-klein-style AIRs; default remains `"sdxl"`.

3. **Flux2Klein sampler set incomplete vs API audit intro** — intro lists more Flux2Klein samplers (`dpm++2s_a`, `dpm++2mv2`, `ipndm`, `ipndm_v`, …); §4b and code only ship four. Fine for this ticket’s table; expand later if the API surface is required.

4. **No unit tests for UI visibility helpers** — `currentEcosystem`, `filteredSamplerPresets`, `maxCFGScale`, `isFieldVisible` are pure enough to unit-test without a TUI. Pre-existing gap (`internal/ui` has zero tests); not introduced as a regression of gates.
