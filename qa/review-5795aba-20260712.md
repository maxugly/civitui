## Findings

### Blocking

1. **Upscale input masking unusable** — `internal/ui/ui.go:688–689` and `:704–705`. Keystroke Validate rejects `val < 320`, so intermediate digits (`"1"`, `"32"`, …) roll back. User cannot enter any valid upscale dimension. Fix: keystroke Validate should match width/height pattern — regex + max only; keep min 320 in `validateNumericFields` on submit.

2. **`isFieldVisible` is dead code** — defined at `internal/ui/ui.go:459` but never called (confirmed repo-wide). Specs require it in:
   - `viewConfig` render loop (`continue` when hidden)
   - tab / shift+tab navigation (skip hidden)
   - `toRequest` (omit hidden optionals)
   Without this, denoise/clipSkip/fluxUltraRaw visibility rules do nothing in the TUI.

3. **Always-on API fields that should be conditional** — `toRequest` (`:753–759`) ignores visibility:
   - Default model is Flux → still sends `clipSkip: 2` (default field value).
   - Non-Ultra flux mode still sends `fluxUltraRaw: false` (non-nil `*bool` is not omitempty-elided).
   Specs explicitly gate these with `isFieldVisible`.

4. **clipSkip model detection incorrect** — `:464–465` lacks `strings.ToLower` and matches `"zImage"` while scheduler code uses `"zimage"` (`:1580`). ZImage models will incorrectly show/send clipSkip even after visibility is wired.

### Non-blocking

1. **No tier-2 marshal tests** — `civit_test.go` has draft coverage but no assertions for `sourceImage`, `denoise`, `clipSkip`, `upscaleWidth/Height`, `experimental`, `fluxUltraRaw` present/omit. Coverage debt; gates still green.

2. **Denoise img2img plumbing incomplete** — no `sourceImage` on Model; denoise hard-hidden via `return false`. Acceptable as staged work only if visibility is wired and denoise stays out of the form until source-image support lands. Today the field still renders (see blocking #2).

3. **Advanced Settings separator missing** — `specs/experimental.md` §3d.

4. **Help text wording** differs slightly from specs (substance OK).

5. **Denoise default value** empty vs spec `"0.4"`.

## Verdict

**FAIL — upscale keystroke min makes upscale fields unusable (`internal/ui/ui.go:688`); `isFieldVisible` never applied to view/nav/toRequest (`:459` vs `:1116`/`:1477`/`:753`); conditional fields leak onto the wire (`clipSkip`, `fluxUltraRaw`).**

Re-run after Tom fixes:

1. Upscale Validate: max-only at keystroke; min only on submit.
2. Wire `isFieldVisible` into `viewConfig`, tab navigation, and `toRequest` (and preferably `validateNumericFields`).
3. Fix clipSkip model check: `ToLower` + `"zimage"`.
4. Optionally add marshal tests for tier-2 tags/omitempty.
